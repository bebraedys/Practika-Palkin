### 3.1.1. Использование выбранной системы контроля версий

**Git** — распределённая система контроля версий, выбранная для данного проекта.

**Обоснование выбора:**
- Распределённое хранение — каждый разработчик имеет полную копию репозитория
- Ветвление и слияние — лёгкое создание веток для новых функций
- Скорость работы — большинство операций выполняются локально
- Надёжность — история версий защищена от потери данных

**Принципы работы Git:**
1. **Инициализация репозитория** — `git init` создаёт локальный репозиторий
2. **Добавление файлов** — `git add <файл>` помещает изменения в индекс
3. **Фиксация изменений** — `git commit -m "сообщение"` сохраняет снимок кода
4. **Ветвление** — `git branch <имя>` создаёт новую ветку разработки
5. **Слияние** — `git merge <ветка>` объединяет изменения из веток
6. **Отправка на сервер** — `git push` загружает изменения в удалённый репозиторий
7. **Получение изменений** — `git pull` загружает и объединяет изменения из сервера

**Структура коммитов в проекте:**
```
feat: добавление управления продажами
fix: исправление ошибки подключения к БД
docs: обновление документации
refactor: рефакторинг кода репозиториев
```

1. **Гибкость** — возможность вносить изменения после каждой итерации

```sql
CREATE SCHEMA app;

-- Таблица типов партнёров
CREATE TABLE app.partner_types (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL UNIQUE
);

-- Таблица партнёров
CREATE TABLE app.partners (
    id SERIAL PRIMARY KEY,
    partner_type_id INTEGER REFERENCES app.partner_types(id),
    name VARCHAR(255) NOT NULL,
    legal_address VARCHAR(500),
    inn VARCHAR(20),
    director_name VARCHAR(255),
    phone VARCHAR(50),
    email VARCHAR(100),
    rating INTEGER DEFAULT 0 CHECK (rating >= 0),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Таблица продаж
CREATE TABLE app.sales (
    id SERIAL PRIMARY KEY,
    partner_id INTEGER REFERENCES app.partners(id) ON DELETE CASCADE,
    product_name VARCHAR(255) NOT NULL,
    quantity INTEGER CHECK (quantity > 0),
    sale_date DATE DEFAULT CURRENT_DATE,
    amount NUMERIC(12,2) CHECK (amount >= 0)
);
```

```csharp
public class PartnerServiceTests
```

### 3.1.7. Интегрирование программных модулей

**Процесс интеграции программных модулей:**

**1. Подготовка к интеграции**
- Все модульные тесты проходят успешно
- Код соответствует стандартам кодирования
- Документация обновлена

**2. Стратегии интеграции**

**Нисходящая интеграция:**
```
Слой представления (WPF)
         ↓
Слой бизнес-логики (Services)
         ↓
Слой доступа к данным (Repositories)
         ↓
База данных (PostgreSQL)
```

**Восходящая интеграция:**
```
База данных → Репозитории → Сервисы → Представление
```

**3. Этапы интеграции в проекте**

**Этап 1: Интеграция репозиториев с БД**
```csharp
// Проверка подключения
var context = new PalkinDbContext(connectionString);
var partners = await context.Partners.ToListAsync();
```

**Этап 2: Интеграция сервисов с репозиториями**
```csharp
// Регистрация зависимостей
services.AddTransient<IPartnerRepository, PartnerRepository>();
services.AddTransient<IPartnerService, PartnerService>();
```

**Этап 3: Интеграция UI с сервисами**
```csharp
// Внедрение сервиса в ViewModel
public MainWindow(IPartnerService partnerService)
{
    _partnerService = partnerService;
}
```

**4. Проблемы интеграции и решения**

| Проблема | Решение |
|----------|---------|
| Конфликты версий зависимостей | Единый файл Directory.Packages.props |
| Несоответствие типов данных | Использование ViewModel для маппинга |
| Ошибки подключения к БД | Централизованная строка подключения в App.xaml.cs |
| Проблемы с потокобезопасностью | async/await для всех операций с БД |

**5. Интеграционное тестирование**
```csharp
[Fact]
public async Task IntegrationTest_AddPartnerAndVerifyInDatabase()
{
    // Arrange
    var connectionString = "Host=localhost;Database=PalkinTest;...";
    await using var context = new PalkinDbContext(connectionString);
    await context.Database.EnsureCreatedAsync();
    
    var service = new PartnerService(
        new PartnerRepository(() => context),
        // ... другие зависимости
    );
    
    // Act
    var partner = new PartnerViewModel { Name = "Test Partner", ... };
    await service.AddPartnerAsync(partner);
    
    // Assert
    var dbPartner = await context.Partners.FirstOrDefaultAsync(p => p.Name == "Test Partner");
    Assert.NotNull(dbPartner);
}
```
